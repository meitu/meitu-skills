# Task ID Baseline

This package does not ship fixed public `task_id` samples.

`task_id` values are generated by the remote runtime, so they vary by account, region, input, and time. For ClawHub and GitHub releases, use this file as a refresh checklist rather than a source of static IDs.

## How To Refresh

1. Install `meitu-cli@2.1.8`.
2. Configure valid Meitu OpenAPI credentials.
3. Run the representative command for the target skill with `--json`.
4. Record one successful `task_id` from your environment for release validation.

## Release Validation Note

When preparing a ClawHub release, it is enough to confirm that each listed skill can return a valid `task_id` in a smoke test. Do not hard-code environment-specific IDs into the public package.
